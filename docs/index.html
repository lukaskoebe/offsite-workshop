Let's get your Mac ready

A quick, copy-paste setup. Install two tools, run one command, and you'll have the app running locally — no prior experience needed.

⏱️ ~15 minutes 💻 macOS (Apple Silicon & Intel) 📋 Just copy & paste
👋

Everything happens in the Terminal app. Each step has a Copy button — paste the command into Terminal and press Return. Work through the steps in order. If anything looks off, check the troubleshooting section at the bottom.

1

Open the Terminal

Press ⌘ + Space to open Spotlight, type Terminal, and press Return. A window with a text prompt will appear — that's where every command below goes.

2

Install Homebrew

Homebrew is the package manager we'll use to install Git and Docker. Paste this and press Return. It may ask for your Mac password (typing stays invisible — that's normal). 

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
🍎 Apple Silicon (M1–M4): when it finishes, Homebrew prints two “Next steps” lines starting with echo. Copy and run those so the brew command is found. (On Intel Macs you can skip this.)
3

Install Git

Git lets you download (“clone”) the workshop project.

brew install git
4

Install Docker Desktop

Docker runs the app and its backend for you — no other setup required. 

brew install --cask docker
🐳 After it installs, open Docker Desktop from your Applications folder once and leave it running. Wait until the whale icon in the menu bar stops animating — that means the engine is ready.
5

Download the project

This copies the workshop repository into a folder on your Mac (in your home directory). 

git clone https://github.com/lukaskoebe/offsite-workshop.git
6

Add your workshop API key

The coding agent needs an access token (you'll be given one for the workshop). Move into the project folder, create your private settings file from the template, and open it: 

cd offsite-workshop
cp example.env .env
open -e .env

A TextEdit window opens showing ASKAI_API_TOKEN=. Paste your token right after the = (no spaces, no quotes) so it reads like ASKAI_API_TOKEN=sk-…. Save with ⌘ + S and close the window.

🔒 Your .env file stays on your machine — it's never shared or uploaded.
7

Start the app

Still in the same Terminal window — you're already in the project folder — start everything. The first run downloads a few things, so give it a couple of minutes. 

docker compose up

When you see the log lines settle and mention VITE ready, the app is up. Leave this window running during the workshop.

🎉

You're all set!

Open your browser and visit the app. Keep the Terminal window open — closing it stops the app.

Open http://localhost:3000
🤖

Meet your coding agent

The setup also starts an AI coding agent (opencode), already opened on this project. That's where you'll build new features during the workshop — just describe what you want and let it code.

Open http://localhost:4096

Uses the API key you set in step 6. It opens the Recipe Book project automatically — no folder to pick.

Troubleshooting

brew: command not found
Homebrew installed, but your shell can't find it yet (common on Apple Silicon). Run the two echo lines Homebrew printed at the end of Step 2, then close and reopen Terminal. If you missed them, run: 
echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zprofile
eval "$(/opt/homebrew/bin/brew shellenv)"
Cannot connect to the Docker daemon
Docker Desktop isn't running. Open it from your Applications folder and wait for the whale icon in the menu bar to stop animating, then run docker compose up again.
Port 3000 is already in use
Another program is using that port. Stop it, or stop the app with Control + C in the Terminal and try again.
How do I stop the app?
Click the Terminal window running the app and press Control + C. To start it again later, run cd offsite-workshop then docker compose up.
